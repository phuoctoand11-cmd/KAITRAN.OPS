Implement employee performance scoring system focused on cleaning staff.

Goal:
Build a soft KPI / scoring / bonus-warning system.
This is NOT a hard punishment system.
Main purpose:
- motivate staff
- improve service quality
- track reliability
- create fair bonus system

====================================================
1. CREATE PERFORMANCE MODULE
====================================================

Create new sidebar page:

"Hiệu suất nhân viên"
(Employee Performance)

Visible for:
- admin
- manager

Cleaning staff can only view their own scores.

====================================================
2. DATABASE DESIGN
====================================================

Create tables:

employee_performance_scores
employee_performance_logs
employee_monthly_reviews

----------------------------------------------------
employee_performance_scores
----------------------------------------------------

Fields:
- id
- employee_id
- month
- year
- total_score (default 100)
- bonus_amount
- penalty_amount
- rank
- warning_level
- notes
- created_at
- updated_at

----------------------------------------------------
employee_performance_logs
----------------------------------------------------

Stores every point change.

Fields:
- id
- employee_id
- task_id nullable
- booking_id nullable
- score_change
- reason
- category
- created_by
- created_at

Categories examples:
- checklist
- lateness
- cleanliness
- customer_feedback
- teamwork
- emergency_support
- violation
- bonus

----------------------------------------------------
employee_monthly_reviews
----------------------------------------------------

Fields:
- id
- employee_id
- month
- year
- strengths
- weaknesses
- improvement_plan
- manager_comment
- created_by
- created_at

----------------------------------------------------

- no complaints for 7 days

====================================================
4. WARNING / PENALTY LOGIC
====================================================

IMPORTANT:
Do NOT punish immediately for every mistake.

Only apply soft penalty when score falls below thresholds.

----------------------------------------------------
Thresholds
----------------------------------------------------

90-100:
- Excellent

80-89:
- Good

70-79:
- Normal

60-69:
- Light warning

50-59:
- Strong warning

40-49:
- Performance deduction

Below 40:
- Review required

----------------------------------------------------
Penalty suggestion
----------------------------------------------------

Below 50:
- small KPI deduction

Below 40:
- manager review

Below 30:
- suspension consideration

IMPORTANT:
This should be configurable from settings.
